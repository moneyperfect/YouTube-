背景

为什么做这个工具

我经常需要翻译 YouTube 上的长视频，但市面上的工具要么太贵，要么质量太差。DeepL 翻译字幕效果一般，专门的视频翻译 SaaS 又太贵。

于是我决定自己做一个。更重要的是，我想验证一件事：用 AI 辅助编程，一个人能走多远？

时间线

72 小时发生了什么

Day 1

核心链路搭建

yt-dlp 视频下载 → faster-whisper 转写 → DeepSeek API 翻译 → ffmpeg 字幕烧录。跑通最小可用版本。

Day 2

产品化重构

前端从 Gradio 迁移到原生 HTML/JS，加入 FastAPI 后端。配置持久化、任务队列、批量处理。

Day 3

架构优化与发布

代码模块化拆分（config/routes/services），iPhone 质感 UI 重写，添加加载动画和 Toast 提示。开源发布。

技术栈

用了什么技术

Python FastAPI yt-dlp faster-whisper ffmpeg OpenAI API 原生 JS Docker

后端用 Python + FastAPI，前端用原生 HTML/CSS/JS（无框架）。翻译支持 5 种 LLM 后端：DeepSeek、OpenAI、通义千问、Moonshot、Ollama 本地。

"AI 时代，技术能力是基本功，产品思维才是分水岭。"

复盘

学到了什么

1. AI 能做什么

代码编写、调试、重构、测试生成、文档撰写——AI 全都能做。但它需要你给出清晰的指令和正确的方向。

2. AI 不能做什么

产品判断、用户体验、商业决策——这些仍然需要人来思考。AI 是执行者，不是决策者。

3. 最大的收获

不是代码本身，而是用 AI 构建产品的思维方式。如何拆解需求、如何描述问题、如何验证结果——这些能力比写代码更重要。

产品

最终成品

一个完整的 YouTube 视频字幕翻译工具，支持：

• 输入链接自动下载视频和字幕
• 多 LLM 后端翻译（DeepSeek、OpenAI 等）
• 字幕编辑器（单句编辑、重译、锁句）
• 视频导出（烧录中文字幕或双语字幕）
• 批量处理、任务队列、配置持久化
