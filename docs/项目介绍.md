# YTSubViewer — YouTube 长视频字幕翻译工作台

## 一句话介绍

粘贴 YouTube 链接，自动下载、转写、翻译，输出双语字幕和硬字幕视频。

---

## 解决什么问题

YouTube 上有大量优质英文长视频（演讲、访谈、技术分享），但对中文观众来说存在语言障碍。现有的浏览器翻译插件只能处理短文本，无法胜任 30-120 分钟长视频的字幕翻译。YTSubViewer 是一个完全独立的本地桌面客户端，专门为长视频场景设计：

- **自动获取字幕**：优先复用 YouTube 人工英文字幕，没有时回退到本地 Whisper 转写
- **高质量翻译**：通过 DeepSeek 等大模型批量翻译，支持术语表、风格预设、质量自检与自动重译
- **一站式输出**：中文字幕 SRT、中文 ASS、双语 ASS、烧录硬字幕 MP4
- **完全本地**：视频、字幕、任务状态全部保存在本地，不依赖任何云服务或外部数据库
- **绿色便携**：构建产物自带 Python 运行时和全部依赖，整个文件夹拷到任意 Windows 电脑即可运行

## 核心功能

### 字幕获取

| 来源 | 说明 |
|------|------|
| YouTube 人工字幕 | 最高优先级，直接下载 VTT 格式 |
| YouTube 自动字幕 | 次优先级，可通过设置切换 |
| 本地 Whisper 转写 | 最后回退方案，使用 faster-whisper 本地运行，模型缓存在项目目录内 |

### 翻译引擎

- 支持多家 LLM 后端：DeepSeek、OpenAI、通义千问、Moonshot、Ollama（本地）
- 4 种翻译风格预设：Default（通用）、Creator（创作者/访谈）、Conference（演讲/会议）、Technical（技术/工程）
- 术语表和保护词：确保专有名词、品牌名、技术术语翻译一致
- 自动质量检测：翻译完成后自动检测保留英文过多的字幕行，触发重译
- 批量并行翻译：多线程并发调用 API，加速长视频处理

### 字幕编辑

- 单句编辑：直接修改任意一条字幕的译文
- 单句重译：对不满意的字幕重新调用 API 翻译
- 锁句：锁定已满意的字幕，防止批量替换影响
- 批量术语替换：一键替换所有字幕中的某个词
- 质量报告：自动检测可能的问题字幕（英文残留、过短、过长、缺失等）

### 导出与播放

- 中文字幕 SRT
- 中文 ASS 字幕（优化断句，适合播放器挂载）
- 双语 ASS 字幕（英文 + 中文上下排列）
- 烧录硬字幕 MP4（中文 / 双语）
- 快速预览版 MP4（低分辨率，适合快速检查）
- 本地 mpv 播放器一键挂载字幕播放

### 批量处理

- 支持多链接批量入队
- 后台任务队列，刷新页面不丢失进度
- 任务历史、失败重试、取消任务
- 频道/创作者级术语和风格记忆

### 运行架构

YTSubViewer 是纯本地单机应用，所有组件运行在同一进程内：

| 组件 | 实现 | 说明 |
|------|------|------|
| Web 服务 | FastAPI + uvicorn | HTTP API 和静态资源服务 |
| 任务队列 | Python threading | 本地并发，最多 3 个并行任务 |
| 数据存储 | JSON 文件 | 任务状态持久化在 `workspace/.runtime/tasks/` |
| 工具链 | ffmpeg + mpv | 内置于 `.tools/` 目录，无需系统安装 |
| 转写模型 | faster-whisper | 模型缓存在项目 `models/` 目录，禁止联网下载 |

### 关键模块

| 模块 | 文件 | 职责 |
|------|------|------|
| 入口 | `app.py` | 启动 uvicorn，自动打开浏览器 |
| Web API | `webapp.py` | FastAPI 路由，所有前端接口 |
| 流水线 | `pipeline.py` | 翻译流程编排器，Generator 模式 |
| 翻译 | `services/translate.py` | OpenAI 兼容 API 调用，质量自检 |
| YouTube | `services/youtube.py` | yt-dlp 封装，Cookie 管理 |
| 转写 | `services/transcribe.py` | faster-whisper 本地转写 |
| 导出 | `services/export.py` | ffmpeg 字幕烧录 |
| 字幕处理 | `subtitle_processing.py` | VTT/SRT/ASS 解析与生成 |
| 后台任务 | `background_jobs.py` | threading 任务队列管理 |
| 配置 | `config.py` | Settings dataclass，便携路径解析 |
| Provider | `providers.py` | LLM 后端注册表 |
| 授权 | `license.py` | 激活码远程验证 + 本地持久化 |

## 快速开始（开发模式）

### 环境要求

- Python 3.11+（仅开发模式需要，便携版自带）
- ffmpeg（已内置于 `.tools/` 目录，无需系统安装）
- mpv（已内置于 `.tools/` 目录，可选）
- NVIDIA GPU（推荐，加速 Whisper 转写，非必须）

### 安装与启动

```powershell
# 克隆项目
git clone <repo-url>
cd 油管翻译

# 创建虚拟环境
python -m venv .venv
.venv\Scripts\Activate.ps1

# 安装依赖
pip install -r requirements.txt

# 启动
python app.py
```

应用会自动选择可用端口并打开浏览器。

### 使用流程

1. 首次打开时，如提示激活，请输入激活码完成授权
2. 粘贴 YouTube 链接 → 点击「分析」
3. 确认视频信息和字幕策略 → 点击「开始翻译」
4. 等待翻译完成（进度实时更新）
5. 在结果页下载字幕文件或导出硬字幕视频

## 构建便携版（Embeddable Python）

YTSubViewer 采用 Windows Embeddable Python 方案实现完全绿色便携，用户无需安装 Python 即可运行。

### 一键构建

```powershell
python build_portable.py
```

脚本自动完成以下流程：

| 步骤 | 操作 | 说明 |
|------|------|------|
| 1 | 准备输出目录 | 创建 `dist/YTSubViewer/` 及子目录 |
| 2 | 下载 Embeddable Python | 从 python.org 下载 3.11.9 嵌入式包（约 10MB），解压到 `runtime/python/` |
| 3 | 配置 pip + 安装依赖 | 修改 `._pth` 启用 site-packages → `ensurepip` 引导 pip → 安装 `requirements.txt` 全部依赖 |
| 4 | 迁移项目组件 | 拷贝 `src/`、`app.py`、`.tools/`、`models/`，创建 `workspace/` |
| 5 | 生成启动器 | `.bat` 启动脚本 + `.vbs` 无窗口启动器 + 使用说明 |

Python 压缩包会缓存在 `.tmp/` 目录，后续构建跳过下载。

### 构建产物结构

```
dist/YTSubViewer/
├── 启动YTSubViewer.bat          # 双击启动（显示控制台）
├── 启动YTSubViewer(后台).vbs    # 无窗口后台启动
├── 使用说明.txt
├── .env.example
├── app.py                       # 应用入口
├── requirements.txt
├── src/                         # 应用源码
│   └── ytsubviewer/
├── .tools/                      # 内嵌工具（如存在）
│   ├── ffmpeg/bin/ffmpeg.exe
│   └── mpv/mpv.exe
├── models/                      # Whisper 模型缓存（如存在）
│   └── distil-large-v3/
├── workspace/                   # 所有运行时数据
│   ├── jobs/                    # 任务产出物
│   ├── .runtime/                # 任务队列状态
│   └── .cache/                  # 模型缓存
└── runtime/
    └── python/                  # 嵌入式 Python 3.11.9
        ├── python.exe
        ├── python311._pth       # 已配置：启用 import site
        ├── python311.zip        # 标准库
        └── Lib/
            └── site-packages/   # 所有 pip 依赖
```

### 关键技术实现

**Embeddable Python 配置**：嵌入式 Python 默认的 `._pth` 文件中 `import site` 被注释掉，构建脚本自动取消注释并添加 `Lib/site-packages` 路径，使 pip 安装的第三方包可被正常导入。

**pip 引导**：通过 `ensurepip --default-pip` 在嵌入式 Python 中安装 pip，然后升级到最新版本，再安装项目依赖。

**路径隔离**：启动脚本通过 `PYTHONPATH`、`YTSUBVIEWER_DATA_ROOT`、`YTSUBVIEWER_RESOURCE_ROOT` 等环境变量确保应用在便携目录内自洽运行，不读写外部路径。

**离线保障**：启动脚本设置 `HF_HUB_OFFLINE=1`，禁止 Whisper 在运行时联网下载模型。

### 使用便携版

构建完成后，将 `dist/YTSubViewer/` 整个文件夹拷到目标电脑：

1. 双击 `启动YTSubViewer.bat` 启动（显示控制台窗口）
2. 或双击 `启动YTSubViewer(后台).vbs` 启动（无窗口，后台运行）
3. 浏览器自动打开，首次使用需配置 API Key

无需安装 Python、ffmpeg 或任何依赖。U 盘拷贝即用。

## 激活授权

YTSubViewer 采用买断制授权：

1. 首次打开应用时，显示全屏激活遮罩
2. 输入激活码 → 向远程验证服务器校验
3. 验证通过后，激活状态加密存储在本地
4. 后续启动优先读取本地状态，无需重复联网

配置远程验证服务器：

```env
YTSUBVIEWER_LICENSE_VERIFY_URL=https://your-server.example.com/api/verify
```

未配置远程服务器时，自动回退到本地 HMAC 签名验证（开发模式）。

## 配置参考

### 核心环境变量

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `DEEPSEEK_API_KEY` | - | DeepSeek API 密钥 |
| `WHISPER_MODEL` | `distil-large-v3` | Whisper 转写模型 |
| `PREFER_AUTOMATIC_SUBTITLES` | `1` | 是否优先使用 YouTube 自动字幕 |
| `MAX_CONCURRENT_TASKS` | `3` | 最大并发任务数 |
| `YTSUBVIEWER_LICENSE_VERIFY_URL` | - | 远程激活验证服务器地址 |

### 性能模式

| 模式 | 转写模型 | 翻译并行数 | 导出编码 | 适用场景 |
|------|---------|-----------|---------|---------|
| fast | fallback 模型 | 4+ | ultrafast | 快速预览、短视频批量 |
| balanced | 主模型 | 3 | superfast | 默认，平衡速度和质量 |
| quality | 主模型 | 2 | medium | 正式交付前的最终成品 |

## 适用场景

- **个人学习**：翻译技术演讲、课程视频，配合双语字幕学习英语
- **内容创作者**：为英文视频添加中文字幕，扩大受众
- **批量处理**：批量翻译频道视频，术语表确保翻译一致性
- **离线环境**：数据全部在本地，Whisper 转写无需联网，适合对隐私有要求的场景
- **即插即用**：U 盘拷贝、网盘同步、公司电脑部署，无需管理员权限
